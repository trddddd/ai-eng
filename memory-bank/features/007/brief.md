# Brief — Сессия повторений со спейсед репетишн

**Issue:** trddddd/ai-eng#14

## Проблема

Пользователь изучает слова, но не знает, насколько хорошо он их запомнил и когда нужно повторить каждое слово снова. Без этого он либо повторяет слова слишком часто (тратит время), либо слишком редко (забывает). Нет механизма, который бы адаптировал расписание повторений под его реальный прогресс.

Дополнительная сложность: алгоритм спейсед репетишн принимает на вход дискретный рейтинг качества ответа. При этом реальное взаимодействие пользователя с карточкой порождает несколько сигналов — правильность ответа, время реакции, количество попыток. Любой маппинг этих сигналов в рейтинг необратим — потерянная информация не восстанавливается. Это ключевое проектное решение, которое должно быть принято осознанно.

## Для кого

Пользователь — человек, изучающий английские слова через упражнения типа cloze (заполнение пропуска в предложении).

## Контекст

В системе уже есть карточки (Card) с привязкой к словам и предложениям. Сейчас отсутствует любая механика, которая отвечала бы на вопросы: «когда показывать карточку снова?» и «насколько хорошо пользователь знает это слово?». Задача возникла как необходимый следующий шаг после создания базового словарного контента.

## Желаемый результат

- Пользователь может пройти сессию повторений: видит карточку, вводит ответ, видит — правильно ли он ответил и какой был верный вариант.
- После каждого ответа система оценивает его качество и на этой основе планирует следующее повторение карточки.
- История ответов сохраняется в системе для использования алгоритмом — без экспозиции пользователю.

## Вне скоупа

- Другие типы упражнений кроме cloze — не сейчас.
- Аналитический дашборд для пользователя — не сейчас.
- Настройка алгоритма пользователем — не сейчас.
