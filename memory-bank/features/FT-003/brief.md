# Brief: Test Coverage & CI Setup

**Issue:** trddddd/ai-eng#4
**State:** CLOSED

## Проблема

Кодовая база растёт, но нет автоматической проверки корректности кода при каждом изменении. Регрессии обнаруживаются не в момент code review, а после merge в main. Нет уверенности, что новые коммиты не ломают существующую функциональность.

## Для кого

Разработчики проекта: нужна уверенность в стабильности кода при каждом pull request и merge в main.

## Откуда взялась задача

Доменный слой content bootstrapped (фича #2). Следующий шаг — выстроить safety net перед тем, как продолжать добавлять новую функциональность: study flow, карточки, аудио.

## Что мы хотим получить

- Существует автоматическая проверка кода при каждом изменении в репозитории
- Покрытие тестами ≥80%
- Статус проверок виден без открытия репозитория

## Вне скоупа

Нагрузочное тестирование, e2e-тесты, coverage-гейты на уровне отдельных файлов, настройка деплоя.
