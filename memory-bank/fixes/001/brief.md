# Brief: Некорректный матчинг слов при импорте предложений

**Issue:** trddddd/ai-eng#17
**State:** CLOSED

## Проблема

Алгоритм матчинга слов использует substring match без учёта границ слов. Пример: слово "ago" матчится внутри "agony", хотя это два разных слова. Это происходит из-за использования `String#include?`, который проверяет вхождение подстроки, а не отдельных слов. При импорте предложений это приводит к некорректной ассоциации лексем с текстом.

## Для кого

Пользователь — человек, который использует инструмент импорта предложений (Quizword corpus). Некорректный матчинг приводит к тому, что предложение связывается не с тем словом, что нарушает качество данных.

## Откуда взялась задача

Личный диалог с пользователем от 2026-04-07. При импорте предложений было обнаружено, что слово "ago" ассоциируется с "agony" в контексте предложения, хотя семантически это разные слова.

## Что мы хотим получить

Алгоритм матчинга слов должен:
- Совпадать только целиком, а не как часть другого слова (например, "ago" не должно совпадать с "agony")
- Быть case-insensitive (например, "Ago" должно совпадать с "ago")

## Вне скоупа

- Поддержка опечаток и нечёткого матчинга
- Учёт словоформ (например, "run" и "running" считаются разными словами)
- Фаззи-поиск по всему корпусу
- Многоязычный матчинг (только английский)
- ML-based semantic similarity
